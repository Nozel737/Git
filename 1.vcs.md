Version Control System(vcs) is a system that keeps changes records to a file over time so that you can recore specific versions later. They are recorded at any time.



types
1)centralised vcs: It's where everyone in a team has direct access to the central server
2)Distributed vcs: It's where every contributor has a local copy or version of the repository.
3)Local vcs: It's where all changes are tracked on your machine with no external collab.



Importance
-collaboration between remote developers is easy.
-Manages versions i.e all versions of code are preserved.
-Help analyse and compare versions.
Help reduce downtime: reverse faulty updates and downtime.



Definition of git: It's a distributed control system that provides and distribute none linear work flows by providing data assurance for deploying quality softwares.
Definition of github: It's a central repository or server where all codes lives or are kept.




Definition of repository: It's a directory or folder or storage space where your projects can be saved. It can be a local folder in your computer or storage space on a remote host online(github)
